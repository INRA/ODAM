Wrapper Functions for ODAM Web Services

Description

ODAM (Open Data for Access and Mining) is an Experimental Data Table Management System (EDTMS) that implements a simple way to make research data broadly accessible and fully available for reuse, including by a script language such as R. The main purpose is to make a dataset accessible online with a minimal effort from the data provider, and to allow any scientists or bioinformaticians to be able to explore the dataset and then extract a subpart or the totality of the data according to their needs.

The Rodam package has only one class, '''odamws''', that provides methods to allow you to retrieve online data using ODAM Web Services. This obviously requires that data are implemented according the ODAM approach , namely that the data subsets were deposited in the suitable data repository in the form of CSV files associated with their metadata also described in CSV files.

See

  • https://www.slideshare.net/danieljacob771282/odam-open-data-access-and-mining for further information.


Requirements

  • RCurl, limma, data.tree (suggested)


Installation

Just install the package in the R environment as shown below:

install.packages("Rodam", repos="http://cran.rstudio.com/")

NOTE:

  • a 'WSEntry' is an alias name associated with an attribute that allows user to query the data subset by putting a filter condition (i.e. a selection constraint) on the corresponding attribute. Not all attributes have a WSEntry but only few ones, especially the attributes within the identifier and factor categories. For instance, the WSEntry of the 'SampleID' attribute is 'sample'. Thus, if you want to select only samples with their ID equal to 365, you have to specify the filter condition as 'sample/365'.

Plot distribution of some variables

Plot distribution of some variables defined in ds$varnames, and fit them as a Weibull distribution

Requirements

  • fitdistrplus

See

  • https://cran.r-project.org/doc/contrib/Ricci-distributions-en.pdf
  • https://cran.r-project.org/web/packages/fitdistrplus/vignettes/paper2JSS.pdf
 library(fitdistrplus)

 # get 'activome' data subset
 ds <- dh$getSubsetByName('activome')

 # Select the level of the factor "Treatment" and the (quantitative) variables
 #SelLines <- TRUE
 SelLines <- ds$data$"Treatment"=='Control'

 vars <- ds$varnames[1:16]
 sqlvars = round(sqrt(length(vars))+0.5)


 par(mfrow=c(sqlvars,sqlvars))
 par(mar=c(4,4,2,1))

 M <- NULL
 for ( myvar in vars) { 
    # Get the subset of data 
      subdata <- ds$data[SelLines, myvar]
    # Plot the distribution of the variable for a specific treatment
      hc <- hist(subdata, probability=T, main=myvar)
      rug(subdata)

    # Fit data with a Weibull distribution, then plot it 
      subdata.wona <- as.numeric(as.vector(na.omit(subdata)))
      fw <- fitdist(subdata.wona,"weibull")
      yfw <- dweibull(hc$mids, shape=fw$estimate[1], scale = fw$estimate[2], log = FALSE)
      lines(density(subdata.wona), lwd = 1, col="blue")
      lines (hc$mids,yfw, lwd = 2, col = "red")
      M <-  cbind( M, fw$estimate )
 }

 refID <- "SampleID"
 subsetList <- c( "samples", "activome", "qNMR_metabo", "cellwall_metabo" )
 n <- length(subsetList)
 Mintersubsets <- matrix(data=0, nrow=n, ncol=n)
 for (i in 1:(n-1))

Compare two levels of the same factor

Based on Hierarchical Clustering Analysis, we plot 2 two dendrograms side by side, each variable being connected on each side. Thus, it allows us to see if some relevant clusters are very different or not depending on the factor level.

Requirements

  • dendextend

See

  • http://www.sthda.com/english/wiki/beautiful-dendrogram-visualizations-in-r-5-must-known-methods-unsupervised-machine-learning
  • https://cran.r-project.org/web/packages/dendextend/vignettes/introduction.html
suppressPackageStartupMessages(library(dendextend))

# set bottom, left, top and right margins 
par(mar=c(2,1,1,8))

refID <- "SampleID"
setName1 <- "activome"
setName2 <- "qNMR_metabo"

dsMerged <- dh$getMerged(refID, setName1, setName2)
treatment1 <- unique(dsMerged[, "Treatment"])[1] # Control
#treatment2 <- unique(dsMerged[, "Treatment"])[2] # WaterStress
treatment2 <- unique(dsMerged[, "Treatment"])[3] # Shaded

ds1 <- dh$getSubsetByName(setName1)
ds2 <- dh$getSubsetByName(setName2)

subdata1 <- dsMerged[dsMerged[, "Treatment"]==treatment1, ][-c(1:4)]
subdata1 <- log2(subdata1)

subdata2 <- dsMerged[dsMerged[, "Treatment"]==treatment2, ][-c(1:4)]
subdata2 <- log2(subdata2)

# Dendrogram 1 : Activome + qNMR_metab for Treatment=Control
dend_01 <- t(subdata1) %>%  scale %>% dist %>% hclust(method = "average") %>% as.dendrogram
cols <- rep( 'red', length(dend_01 %>% labels))
cols[ dend_01 %>% labels %in% ds1$varnames ] <- 'blue'
dend_01 <- dend_01 %>% set("branches_k_color", k = 10) %>% set("labels_colors", cols) %>% set("labels_cex", 0.8)

# Dendrogram 1 : Activome + qNMR_metab for Treatment=WaterStress
dend_02 <- t(subdata2) %>% scale %>% dist %>% hclust(method = "average") %>% as.dendrogram
cols <- rep( 'red', length(dend_02 %>% labels))
cols[ dend_02 %>% labels %in% ds1$varnames ] <- 'blue'
dend_02 <- dend_02 %>% set("branches_k_color", k = 10) %>% set("labels_colors", cols) %>% set("labels_cex", 0.8)


# Comparing two dendrograms
dend12 <- dendlist(dend_01, dend_02)
dend12 %>% untangle(method = "step1side") %>% tanglegram(common_subtrees_color_branches = TRUE, margin_inner=6, main_left=treatment1, main_right=treatment2, cex_main=2)
